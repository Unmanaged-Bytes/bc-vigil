Administration

Backup / Restore / Reset

Admin tab. Three actions:

  • Backup: download a .tar.gz with the SQLite DB, integrity digests and dedup JSON reports. A PRAGMA wal_checkpoint(TRUNCATE) is forced first so the archived DB file is self-contained.
  • Restore: upload a previously produced archive. Replaces DB + digests + dedup directory + trash. An automatic snapshot is taken first.
  • Reset: wipe the DB and all related files (digests, dedup reports, trash). An automatic snapshot is also taken first.

The last two are blocked as long as an integrity or dedup scan is pending/running. Schedulers are stopped during the operation and restarted afterwards.

Concurrency (WAL)

The SQLite DB is opened with journal_mode=WAL, synchronous=NORMAL, busy_timeout=5000. This lets UI reads stay snappy while large writes are flushed (duplicate groups persistence at the end of a big scan).

The bc-vigil.sqlite-wal and bc-vigil.sqlite-shm files are created automatically by SQLite — this is normal and expected in WAL mode. Backup calls an explicit checkpoint so only the main file is archived.

Observability

Two unauthenticated endpoints, designed for external monitoring (Prometheus, Docker healthcheck, curl in cron):

  • GET /health — JSON with status, version, db, scheduler_integrity, scheduler_dedup. Returns 200 when everything is green, 503 when the DB is unreachable or a scheduler is stopped.
  • GET /metrics — Prometheus text exposition. Exposes bc_vigil_up, bc_vigil_info{version}, bc_vigil_db_up, bc_vigil_scheduler_up{module}, bc_vigil_scans_total{module, status}, bc_vigil_dedup_deletions_total{status}. Recommended scrape interval: 30s.

On service startup, scans left in pending or running by a previous dead process are auto-marked failed with the message "interrupted by service restart". An INFO log line reports the cleaned count. A SQLite VACUUM job runs on the 1st of each month at 04:00 (local timezone) to reclaim space after retention purges.

Storage module

The Storage tab shows per-mount disk usage: total, used, free, filesystem type. Informational only, no impact on scans.
